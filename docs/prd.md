# NewsHub 需求文档

## 1. 网站概述

### 1.1 网站名称
NewsHub

### 1.2 网站描述
一个聚合22个主流平台实时热点数据的新闻浏览平台，通过集成hot_news API为用户提供集中、高效的热点新闻阅读体验。\n
### 1.3 网站功能
- 聚合22个主流平台热点内容（百度、微博、知乎、36氪等）\n- 实时热点发现和排序
- 流畅的新闻浏览体验
- 支持按平台和分类浏览
- 零爬虫维护，使用hot_news API获取数据
- 管理后台配置平台参数
- 热度算法计算和内容排序
- 搜索和分类标签功能
- 新闻展示和详情页面

## 2. 平台数据源

### 2.1 API接口
- API地址：https://orz.ai/api/v1/dailynews/\n- 支持22个平台数据获取
- 每30分钟自动刷新一次
- 可配置平台启用/禁用状态

### 2.2 平台列表
**社交媒体**：微博、知乎、抖音、豆瓣、哔哩哔哩
**科技媒体**：36氪、少数派、掘金、V2EX、GitHub Trending、Stack Overflow、Hacker News、吾爱破解
**财经媒体**：新浪财经、东方财富、雪球、财联社
**综合媒体**：百度、今日头条、腾讯网
**其他**：虎扑、百度贴吧

## 3. 核心功能

### 3.1 数据处理
- API数据获取和解析
- 数据清洗和去重
- 内容格式标准化
- URL验证和敏感词过滤
- 数据存储和快照管理

### 3.2 热点算法
- 基于API score的热度计算
- 平台权重和时间衰减因子
- 综合评分公式：`final_score = api_score * platform_weight * time_factor`\n- 多种排序策略

### 3.3 分类与标签
- 平台分类系统（社交、科技、财经等）\n- 基于标题的关键词提取
- 手动标签管理和推荐

### 3.4 搜索功能
- 关键词搜索和高级筛选
- 搜索建议和自动补全
- 热门搜索词统计

### 3.5 新闻展示
- 首页热点轮播和导航
- 列表/卡片两种展示模式
- 无限滚动和分页加载
- 新闻详情页和相关推荐
- 历史趋势查看

### 3.6 用户功能（可选）\n- 个性化推荐
- 收藏和关注
- 用户账户管理

### 3.7 管理后台
- 仪表盘数据统计
- 内容管理和审核
- 平台配置和监控
- API调用状态监控

## 4. 性能与技术要求

### 4.1 性能指标
- 首页加载时间：<2秒
- 新闻列表加载：<1秒
- 搜索响应时间：<1秒
- 新闻详情加载：<1.5秒
- 支持1000+并发用户

### 4.2 技术要求
- 模块化设计，支持横向扩展
- CDN缓存和Redis缓存
- 数据库索引和缓存机制
- 自动化定时任务

### 4.3 安全要求
- 防SQL注入、XSS、CSRF攻击
- API访问频率限制
- 敏感数据加密存储
- HTTPS加密传输

### 4.4 兼容性要求
- 浏览器支持：Chrome、Firefox、Safari、Edge（最新两个版本）\n- 移动端适配：iOS 12+，Android 8+\n- 屏幕分辨率：320px - 4K

## 5. 数据流与架构

### 5.1 数据获取流程
```\n定时任务触发（每30分钟）\n    ↓\n遍历22个平台
    ↓\n调用 hot_news API
    ↓\n接收 JSON 数据
    ↓\n数据清洗和去重
    ↓\n存储到数据库
    ↓\n更新缓存和索引
```\n
### 5.2 用户访问流程
```\n用户请求
    ↓\nCDN 缓存检查（如有）\n    ↓\nRedis 缓存查询
    ↓\n数据库查询（缓存未命中）\n    ↓\n结果聚合和排序
    ↓\n返回展示
```\n
## 6. 法规与风险管理

### 6.1 法律合规
- 遵守《网络安全法》\n- 遵守版权法，注明数据来源
- 跳转至原始平台时需遵循原平台的使用条款
- 用户协议和免责声明

### 6.2 技术风险应对
- API健康监控和降级策略
- 数据缓存和错误重试机制
- 多数据源备份计划
- 网络超时处理

## 7. 设计风格

### 7.1 配色方案
主色调采用深蓝色（#1e3a8a）搭配白色背景，辅以橙色（#f97316）作为强调色，营造专业可信的新闻阅读氛围

### 7.2 视觉细节
采用现代扁平化设计，卡片式布局带有轻微阴影效果，圆角半径8px，图标使用线性风格保持简洁统一

### 7.3 整体布局
首页采用响应式网格布局，新闻列表使用卡片式排列，详情页采用左右分栏设计，确保信息层次清晰易读
